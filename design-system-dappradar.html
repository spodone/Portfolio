    Overview

    When I joined DappRadar, the product had no shared component library — every feature was built from scratch, with inconsistent colors, spacing, and interaction patterns across screens. I led the design and rollout of a token-based design system that became the foundation for every product surface the team shipped.

    The challenge

    DappRadar's product moved fast — new features shipped weekly across Rankings, Portfolio, Staking, and NFT verticals. Without a shared language, designers and developers spent significant time re-solving the same problems. Visual inconsistency was visible to users. Design debt was slowing velocity.

    "Inconsistency isn't just a design problem — it erodes user trust one misaligned button at a time."

    The system

    I built a token-based design system in Figma with variables covering color, typography, spacing, radius, and elevation. A core set of ~30 tokens controlled the entire product's visual language — making global changes a single operation rather than a hunt-and-replace across dozens of files.

    TokensColor, typography, spacing, radius, and elevation — all as named variables mapped 1:1 to CSS custom properties in the codebase
    Components60+ components across buttons, inputs, tables, cards, navigation, and data visualizations — each with all states documented
    PatternsReusable page-level patterns for empty states, loading skeletons, error boundaries, and onboarding flows
    DocsUsage guidelines, do/don't examples, and handoff specs embedded directly in Figma — no separate documentation site needed

    Results

    −60%Reduction in per-feature design time after system adoption
    60+Reusable components shipped in the first version
    3Product verticals unified under one visual language
    ∞Faster designer onboarding — new hires productive from day one
